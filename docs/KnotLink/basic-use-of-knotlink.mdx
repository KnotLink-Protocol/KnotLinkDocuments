# 在工程中使用 KnotLink

## 1 询问者(OpenSocketQuerier)

### 1.1 引入库

在导入 KnotLink 库文件后，可以使用以下代码导入KnotLink标准库，其中，Python支持引入单角色，KnotBlock无须引入。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`from KnotLinkClient_PyQt import OpenSocketQuerier`}
  python={`from KnotLinkClient_Py import OpenSocketQuerier`}
  cpp={`#include "OpenSocketQuerier.hpp"`}
  java={``}
  js={``}
  lua={``}
/>


### 1.2 创建对象

通过OpenSocketQuerier构造函数创建一个询问者，你可以在此时直接传入回复者的appID及openSocketID，也可后续设置。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`querier = OpenSocketQuerier("0x00010000", "0x00000010")`}
  python={`querier = OpenSocketQuerier("0x00010000", "0x00000010")`}
  cpp={`OpenSocketQuerier *querier = new OpenSocketQuerier(); // 在后续设置回复者的appID及openSocketID`}
  java={``}
  js={``}
  lua={``}
/>


### 1.3 发起询问

#### 1.3.1 同步询问

通过成员函数query可对设定的回复者发起同步式询问，线程将被阻塞直到收到回复。此函数返回值为回复的字符串。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`result = querier.query_l("getApplicationStatus")`}
  python={`result = querier.query("getApplicationStatus")`}
  cpp={`std::string result = querier->query_l("getApplicationStatus");`}
  java={``}
  js={``}
  lua={``}
/>

#### 1.3.2 异步询问

##### 1.3.2.1 完全解耦的异步询问

通过成员函数 query_u 可对设定的回复者发起完全解耦的异步式询问。你需要设置回调（或槽）函数处理回复字符串。此函数不会阻塞线程，无返回值。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`def on_data_received(data: str):
      print("接收到数据（字符串）：", data)
  querier.dataBack.connect(on_data_received)`}
  python={``}
  cpp={``}
  java={``}
  js={``}
  lua={``}
/>

:::warning[解耦性]
完全解耦的异步询问意味着你不能确认收到的回复对应哪次询问，除非你的询问频率很低或你在询问字符中加入“标签”
:::

## 1 信号发送者(SignalSender)

信号发送者（SignalSender）负责生成和发送信号。你可以将其想象为一个广播员，他通过麦克风向所有人广播消息。

### 1.1 最小片段示例

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={``}
  python={`from KnotLinkClient_Py import SignalSender
signal_sender = SignalSender(APPID="0x00100000", SignalID="0x00000010")
signal_sender.emitt("Hello,I'm SignalSender!")`}
  cpp={`#include "SignalSender.hpp"
SignalSender *signalSender = new SignalSender();
signalSender->setConfig("0x00100000","0x00000010");
signalSender->emitt("Hello, World!");`}
  java={``}
  js={``}
  lua={``}
/>

### 1.2 使用步骤

#### 引入库

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`from KnotLinkClient_PyQt import SignalSender # 引入 SignalSender`}
  python={`from KnotLinkClient_PyQt import SignalSender # 引入 SignalSender`}
  cpp={`#include "SignalSender.hpp"`}
  java={``}
  js={``}
  lua={``}
/>

#### 创建 SignalSender

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`# 创建一个 SignalSender 实例
signalSender = SignalSender()`}
  python={`# 创建一个 SignalSender 实例
signalSender = SignalSender()`}
  cpp={`SignalSender *signalSender = new SignalSender();`}
  java={``}
  js={``}
  lua={``}
/>

#### 配置 SignalSender

在发送信号之前，你需要配置 SignalSender 的 **APPID** 和 **SignalID**。这些是用于标识信号的唯一标识符。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`# 配置 APPID 和 SignalID
signalSender.setConfig(appID="0x00100000", signalID="0x00000010")`}
  python={`# 配置 APPID 和 SignalID
signalSender.setConfig(appID="0x00100000", signalID="0x00000010")`}
  cpp={`signalSender->setConfig("0x00100000","0x00000010");`}
  java={``}
  js={``}
  lua={``}
/>

#### 发送信号

一旦配置完成，你就可以发送信号了。发送信号时，你需要提供一个字符串格式的数据。

<MultiLangCodeTabs
  qtcpp={``}
  pyqt5={`# 发送信号
signalSender.emitt(data="Hello, World!")`}
  python={`# 发送信号
signalSender.emitt(data="Hello, World!")`}
  cpp={`signalSender->emitt("Hello, World!");`}
  java={``}
  js={``}
  lua={``}
/>
